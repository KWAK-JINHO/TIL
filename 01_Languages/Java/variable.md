# 변수

변수란 값을 저장할 수 있는 메모리상의 공간. 이 공간에 저장된 값은 변할 수 있다.

## 변수의 초기화

1. 인스턴스 변수: Heap 할당, JVM이 자동 초기화.(cause: Heap은 공유자원, 안전성 위해 0,null 등으로 초기화)
2. 로컬 변수: stack 할당, JVM이 자동 초기화하지 않는다. 초기화하지 않고 사용시 컴파일 에러발생

## 변수의 타입

1. 기본형과 참조형

- 기본형 변수: 실제 값으로 boolean, char, byte, short, int, long, float, double 8가지
- 참조형 변수: 값이 저장되어 있는 주소(Heap의 위치)로 위 8가지를 제외한 모든 변수, null을 가질 수 있다.
- C언어처럼 참조형 변수간의 연산은 불가

## 데이터 타입의 선택

- 정수형(= int): CPU가 메모리 주소 체계(32/64bit)에 맞춰 데이터를 가져오는 단위가 일치하는 int가 연산 효율이 높다.(1byte씩 읽는 추가 비용 없음)
- 실수형(= double): float보다 정밀도가 2배 높고, 8byte이기 때문에 64bit 환경에서 표준으로 사용

효율적인 실행보다 메모리를 절약하려면 byte, short를 선택할 수도 있다.

## 타입의 불일치

리터럴 타입이 변수의 타입과 달라도 값의 저장범위가 좁다면 저장이 허용

예시) `int i = 'A'` 문자 A는 2byte로 4byte인 int에 저장된다.('A'는 유니코드 65로 i에 65가 저장)
