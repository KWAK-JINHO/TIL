# Thread Lock

### Thread Lock 이란?

멀티스레드 환경에서 여러 스레드가 공유 자원(메모리, 객체)에 대한 상호 배제(mutual exclusion)를 보장하기 위한 동기화 메커니즘.

### 주요 특징

- 프로세스 내 스레드 간 동기화 (메모리 공유)
- 락 획득 실패 시 대기 전략 선택 가능 (blocking/non-blocking)
- 데드락 발생 가능성 고려 필요
- JVM 레벨에서 최적화 지원 (편향 락, 경량 락)

---

## synchronized (모니터 락)

Java의 기본 동기화 메커니즘으로, 모든 객체가 내재적으로 가진 모니터 락을 활용한다.

### 구현

- 메서드 또는 블록 단위로 임계영역(멀티 프로세스나 멀티 스레드 환경에서 여러 실행 단위가 동시 접근해서는 안되는 영역) 설정
- 진입 시 자동으로 락 획득, 종료 시 자동 해제
- 재진입 가능(Reentrant) - 같은 스레드는 여러 번 획득 가능
- 락 획득 실패 시 BLOCKED 상태로 대기

```java
// 메서드 동기화
public synchronized void method() {
}

// 블록 동기화
synchronized(object){
        // 임계 영역
        }
```
